1. native api - kamera - korisnik uslika pa mu pretvori sliku u crno bijelu i nudi mu download
2. installable - okej
3. cache strategija - cache first - imamo neke pozadinske slike koje su spremljene u cache, ako nisu, server - koristen reactov template - automatski napravi precache od datoteka koje se naprave u buildu, a onda jos ceka zahtjeve za sve .jpg file-ove (osim pozadinske slike neba) i sprema ih u caches
4. offline rad - pretvaranje slika u crnobijele ce radit i offline, jedino mozda pozadinske slike se nece loadati - moze se demonstrirati tako da se instalira aplikacija, otvori, ugasi internet te zatim ode na /ideas preko poveznice
5. background sync - ako je korisnik probao dohvatiti pozadinske slike, ali je offline, slike ce se dohvatiti kada dobije konekciju i spremiti u cache
6. push notifikacija - nista
7. progressive enhancement - ako korisnikov browser ne podrzava service workere, offline rad nece biti moguc ali ce normalno raditi online
